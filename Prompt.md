# PROMPT.md

This document contains some of the prompts and questions used during the development of this project.

---

## Understanding the Requirements

- What are the Functional and Non-Functional Requirements of this project?
- Explain the difference between Functional Requirements and Non-Functional Requirements with examples.
- What is Telemetry Simulation? How can it be implemented in a React application?
- How should I simulate an analytics event without using a real analytics service?
- How can I simulate slow internet or loading states in React?
- What are ARIA labels, and why are they important for accessibility?
- How can I improve keyboard accessibility in React applications?

---

## UI & UX

- How can I make the interface look more like an enterprise dashboard?
- Which color palette works well for healthcare applications?
- How can I improve spacing and typography?
- What are good hover effects for cards and buttons?
- How can I design a better empty state?
- How should I design a professional footer?

---

## Accessibility

- Which HTML elements should have ARIA labels?
- How do screen readers use ARIA labels?
- What are accessibility best practices in React?
- How can I improve Lighthouse Accessibility score?

---

## Performance

- How can I reduce unnecessary component re-renders?
- What are React rendering best practices?
- How can I optimize state updates?

---

## Final Review

- Review my project against the client requirements.
- What improvements can make the UI look more professional?
- Which features would make this project portfolio-ready?
- Does this project follow React best practices?
